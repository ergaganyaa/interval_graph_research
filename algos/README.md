# Эмпирический бенчмарк стратегий кратчайшего пути в графах с интервальными весами ребер

[cite_start]Этот проект представляет собой комплексную среду (**Benchmark Engine**) для объективного сравнения алгоритмов поиска кратчайшего пути в условиях неопределенности[cite: 441, 446]. [cite_start]В отличие от классических обзоров литературы, данный инструмент позволяет тестировать различные математические парадигмы на идентичных наборах данных в реальном времени[cite: 593].

## Команда проекта
* [cite_start]**Болашак Кульмухамбетов** [cite: 205]
* [cite_start]**Ержан Амангельдин** [cite: 205]

## О проекте
[cite_start]В реальных логистических и транспортных сетях точное время в пути (вес ребра) редко бывает известно заранее[cite: 215, 216]. [cite_start]Мы моделируем это как интервал $[L, R]$[cite: 217, 219]. Проект исследует эффективность различных алгоритмов в поиске оптимального пути, когда истинное значение веса становится известно только после принятия решения.

### Ключевая инновация: "Ground-Truth Oracle"
Для обеспечения научной точности мы используем протокол оракула:
* [cite_start]При генерации графа для каждого ребра создается скрытая реализация $x \in [L, R]$[cite: 582, 584].
* [cite_start]Алгоритмы принимают решение ("вслепую"), видя только интервалы $[L, R]$[cite: 585].
* [cite_start]Эффективность алгоритма (Regret) оценивается путем сравнения его пути с идеальным путем, вычисленным оракулом на основе скрытых весов $x$[cite: 591, 602].

## Реализованные парадигмы и алгоритмы
[cite_start]Система объединяет три основные семейства стратегий[cite: 209, 255]:

1.  [cite_start]**Robust Optimization (Karasan / Montemanni):** Минимизация максимального сожаления (Minimax Regret) через MILP-модели[cite: 259, 588].
2.  [cite_start]**Crisp Conversion (Xu / Ebrahimnejad):** Преобразование интервалов в детерминированные веса (среднее значение, индексы приемлемости)[cite: 264, 588].
3.  [cite_start]**Uncertainty-theoretic (Broumi / Moghanni):** Использование нечеткой (Fuzzy), нейтрософской и грубой (Rough) логики[cite: 268, 269].

## Архитектура системы
[cite_start]Проект разделен на три логических слоя[cite: 625, 626]:
* [cite_start]**Data Layer (`generator.py`):** Генерация 1000 синтетических графов (от 3 до 10 000 вершин) в 10 масштабируемых группах[cite: 448, 580, 627].
* [cite_start]**Solver Layer (`algos/`):** Реализация алгоритмов на Python (используются `heapq` для Дейкстры и `Google OR-Tools` для робастных моделей)[cite: 588, 633].
* [cite_start]**Analytics Layer (`analytics.py`):** Сведение результатов в CSV, расчет метрик и построение графиков[cite: 591, 637].
